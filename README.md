# Aplikácia pre správu cestovných príkazov

## Informácie o úlohe
- **Názov úlohy:** Vytvorenie aplikácie pre správu cestovných príkazov  
- **Typ dokumentu:** Zadanie pre programátora  
- **Spracoval:** Branislav Mazúr  

---

## Všeobecný popis
Cieľom úlohy je vytvorenie jednoduchej aplikácie pre správu cestovných príkazov vo veľmi
obmedzenom rozsahu s dôrazom na správny návrh dátových štruktúr a prácu s dátovou vrstvou.  
Pre formu užívateľského rozhrania nie sú definované žiadne požiadavky a nebude predmetom hodnotenia.

---

## Základné entity

### Zamestnanec
- Jednoznačný identifikátor: osobné číslo (reťazec, max. dĺžka 10, nemusí obsahovať iba číslice)  
- Krstné meno  
- Priezvisko  
- Dátum narodenia  
- Rodné číslo  

### Mesto
- Názov mesta (obce)  
- Štát  
- Zemepisné súradnice (šírka, dĺžka)  

### Cestovný príkaz (CP)
Primárna entita definujúca pracovnú cestu. Musí obsahovať:
- Jednoznačný identifikátor CP (celé číslo, generované systémom)  
- Dátum vytvorenia CP (generovaný systémom)  
- Účastník pracovnej cesty (záznam zo zamestnancov)  
- Miesto začiatku pracovnej cesty (záznam z miest)  
- Miesto konca pracovnej cesty (záznam z miest)  
- Dátum a čas začiatku pracovnej cesty  
- Dátum a čas konca pracovnej cesty  
- Doprava: 1 až N položiek z možností:
  - Služobné auto  
  - Autobus  
  - MHD  
  - Pešo  
  - Vlak  
  - Taxi  
  - Lietadlo  
- Stav: jedna z možností
  - Vytvorený  
  - Schválený  
  - Vyúčtovaný  
  - Zrušený  

---

## Funkčné požiadavky
Aplikácia musí:
- Zobraziť na úvodnej stránke zoznam evidovaných cestovných príkazov  
- Umožniť filtrovanie podľa účastníka pracovnej cesty  
- Poskytnúť UI pre:
  - Vytvorenie nového CP  
  - Editáciu existujúceho CP  
  - Vymazanie existujúceho CP  

---

## Technické požiadavky

| Platforma          | .NET           |
|--------------------|----------------|
| Databázový systém  | MS SQL Server  |
| ORM                | nepoužívať     |

Ďalšie požiadavky:
- Súčasťou riešenia musí byť SQL script pre vytvorenie dátovej štruktúry v databáze  
- Script musí obsahovať aj demo záznamy v evidenciách Mesto, Zamestnanec, príp. pridružených entitách (min. 3 záznamy v každej evidencii)  
- Forma frontend aplikácie nie je definovaná (webová aplikácia / WinForm)

# Application for Managing Travel Orders

## Task Information
- **Task Name:** Creation of an application for managing travel orders  
- **Document Type:** Assignment for programmer  
- **Prepared by:** Branislav Mazúr  

---

## General Description
The goal of this task is to create a simple application for managing travel orders in a very limited scope, with emphasis on correct data structure design and working with the data layer.  
The form of the user interface is not defined and will not be evaluated.

---

## Core Entities

### Employee
- Unique identifier: personal number (string, max length 10, may contain non-numeric characters)  
- First name  
- Last name  
- Date of birth  
- National ID number  

### City
- City name (municipality)  
- Country  
- Geographic coordinates (latitude, longitude)  

### Travel Order (CP)
Primary entity defining a business trip. Must include:
- Unique CP identifier (integer, auto-generated by the system)  
- CP creation date (auto-generated by the system)  
- Participant of the business trip (Employee record)  
- Start location of the business trip (City record)  
- End location of the business trip (City record)  
- Start date and time of the business trip  
- End date and time of the business trip  
- Transportation: 1 to N items from the following options:
  - Company car  
  - Bus  
  - Public transport  
  - Walking  
  - Train  
  - Taxi  
  - Airplane  
- Status: one of the following
  - Created  
  - Approved  
  - Settled  
  - Cancelled  

---

## Functional Requirements
The application must:
- Display a list of recorded travel orders on the homepage  
- Allow filtering by participant of the business trip  
- Provide UI for:
  - Creating a new CP  
  - Editing an existing CP  
  - Deleting an existing CP  

---

## Technical Requirements

| Requirement        | Value          |
|--------------------|----------------|
| Platform           | .NET           |
| Database System    | MS SQL Server  |
| ORM                | Not allowed    |

Additional requirements:
- The solution must include an SQL script for creating the database schema  
- The script must also insert demo records into City, Employee, and related entities (minimum 3 records per entity)  
- The form of the frontend application is not defined (developer may choose web app or WinForms)
